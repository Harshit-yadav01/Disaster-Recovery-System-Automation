DR Operations

Guided disaster-recovery lifecycle for the Intern_Automation group. Each action unlocks only when the live array state makes it valid, so the right next step is always the one you can click. Keep Dry-run on to preview the exact commands without changing anything.

Checking array state…
1

Failover

Stop replication on the Primary group, then promote the DR array to Primary (Read/Write). No health check is performed on the Primary — ensure it is failed or inaccessible before executing.

1b

Present to Host

Export the failed-over group's volumes to a DR ESXi host so the recovery VMs can be powered on. Only this group's own volumes are exported. Pick the host below (defaults to the configured target).

After a failover, choose one path
2a

Revert Failover

Discards the DR site's changes and returns to the original Primary in one step (setrcopygroup reverse -local -current). Use when the DR writes made during the outage are not to be kept. This cannot be undone.

OR
2b

Reverse Sync

Keeps the DR site's changes: reverse replication and copy them back to the original Primary (recover → sync), waiting until every volume is synchronized. Then continue to Restore.

3

Restore

Only after a Reverse Sync (2b). Hand control back: return to the natural direction with the Primary Read/Write (source) again and the DR array Read-Only (target).
